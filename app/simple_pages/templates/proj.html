Git

Branch

In Git, a branch is analogous to a tree branch. A tree branch is connected to the trunk, which is the center section of the tree. While branches can grow and fall off, the trunk remains compact and is the only portion of the tree that can be said to be alive and well. Similarly, a Git branch allows you to continue creating and coding a new feature or software modification without affecting the rest of the project. Branches can also be thought of as a new line of development in a project. The master branch is Git's default or primary branch (similar to a trunk of the tree). The main branch is created at the same time as the repository (or the default branch).

Commit

The term "commit" is used in Git to refer to the act of saving changes. Git does not automatically add modifications to a commit. Before performing the Git commit command, you must specify which file and modifications need to be preserved. The commit command only saves changes in Git's local repository, not on remote servers.

Merge

To merge modifications from another branch, use the "merge" command. The current checked out HEAD branch is always the integration's target (i.e. the branch that gets modifications). While Git can handle the majority of integrations automatically, some modifications will cause conflicts that the user must resolve. In our online book, Merge Conflicts, you can learn more about how to deal with them. The master branch is Git's default or primary branch (similar to a trunk of the tree). The main branch is created at the same time as the repository (or the default branch).

Tags

The git tag command and the Git idea of tagging will be discussed in this paper. Tags are a type of remark that refers to a specific point in Git's history. Tagging is a method of capturing a certain point in time for a specific version release (i.e. v1.0.1). A tag is similar to a non-changing branch. Tags, unlike branches, have no history of commits after they are formed. Visit the git branch page for additional information about branches. This page will go over the many types of tags, how to create tags, how to list all tags, how to delete tags, how to share tags, and more.

Repository

The repository in Git is similar to a VCS data structure for storing metadata for a set of files and folders. It holds both the file collection and the history of changes made to those files. In Git, a repository is equivalent to a project folder. All project-related data is stored in a repository. Different projects have their own repositories.
